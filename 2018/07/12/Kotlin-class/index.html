类

可见性和类修饰符

 在 Java 中，我们会为属性设置可见性；会为每个类设置是否允许被继承的权限，在 Kotlin 中也有这样的规则。类中的访问修饰符有：

修饰符对应的成员备注
final不能被覆盖类成员的默认修饰符
open可以被覆盖必须显式的指定
abstract必须被覆盖只能在抽象类中使用，抽象成员不能有实现
override在一个子类中覆盖一个成员如果没有被标记为 final ，覆盖的成员默认是开放的。

 可见性修饰符为：

修饰符对应的成员
public(默认可见性)所有地方可见
internal模块内可见
protected子类内部可见
private类内部可见
