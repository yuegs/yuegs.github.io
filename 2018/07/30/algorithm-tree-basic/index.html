 在 归并排序算法 中，实际上我们就看到了一个二叉树。而 sort 部分，更是一个对树的遍历 。树这个概念，应用也十分广泛，特别适用于层级结构。在开发过程中，我们会遇到 View 树，文件树，组织结构树等等。

二叉树的基本概念

 二叉树还是一颗树。所以，我们先从 树 开始。

树的基本概念

 树，是由顶点（结点）和边构成的数据结构。没有结点的称之为空树。对于一个非空的树，包含了根结点和由结点组成的层次结构。

 如上图所示，先介绍几个基本概念：

  • 根结点：树的最顶端的结点
  • 孩子结点和父结点：两个相连的结点，远离根结点的称为孩子结点，反之称为父结点
  • 边：两个结点之间的连线
  • 兄弟结点：同一个父结点下的所有直连子结点
  • 叶子结点：没有子结点的结点
  • 路径：连接一个结点和它后代结点之间的结点和线
  • 祖先结点：从根到该结点所经分支上的所有结点
  • 结点的深度：对于任意结点n,n的深度为从根到n的唯一路径长，根的深度为0
  • 结点的高度：对于任意结点n,n的高度为从n到一片树叶的最长路径长，所有树叶的高度为0
  • 树的高度：根结点的高度
  • 结点的度：对于一个给定的结点，它子结点的个数就是它的度。显然，叶子结点的度为0

 需要注意的是，有的书中，将根结点算作是第一层。

二叉树

 二叉树，是一种特殊的树，它的子结点个数不超过两个。其中，左边的子树为左子树，右边的子树为右子树，不能颠倒顺序。
 如果一个二叉树，每个父结点都有两个子结点，且所有叶子结点都在同一层，那么，我们称这棵树为满二叉树。
 对于一个有 h 层的满二叉树，结点的个数为 [21+22+…+2h] = 2h+1 - 1。（注意，此处 h 可以为0，为 0 时表示根结点）
 假设二叉树的深度为h，除第 h 层外，其它各层 (1～h-1) 的结点数都达到最大个数，第 h 层所有的结点都连续集中在最左边，这就是完全二叉树。
 完全二叉树中结点的个数为 2h 到 2h+1 - 1 之间。

遍历

 为了能够对树结构进行处理，遍历是一种非常基础的机制。二叉树的遍历，最简单的方式就是使用递归。当然，也可以借助其他工作，使用非递归的方式进行。

前序遍历

 前序遍历定义如下：

  • 访问根结点
  • 按前序遍历的方式遍历左子树
  • 按前序遍历的方式遍历右子树

中序遍历

 中序遍历定义如下：

  1. 遍历左子树，如果左孩子不为空，重复步骤 1，直到左孩子为空
  2. 访问根结点，注意，该根结点是相对于子树而言
  3. 如果右孩子为空，重复步骤2，否则，将右孩子作为根结点，重复步骤1

 或者，以递归的方式来定义：

  1. 以中序遍历的方式遍历左子树；
  2. 访问根节点
  3. 以中序遍历的方式遍历右子树

层次遍历

 层次遍历定义如下：

  • 访问根结点
  • 按照从左到右的顺序依次访问 h 层结点，并将 h+1 层结点依次入队列
  • 重复上述操作

深度优先遍历

 深度优先遍历定义为：

  • 访问根结点
  • 如果有左子树，优先访问左子树，然后访问右子树
  • 重复上述步骤

 我们已经看到，层次遍历借助队列完成；深度优先遍历借助栈完成。

二叉树重建

 在没有重复元素的情况下，根据前序遍历结果和中序遍历结果，是可以重建原二叉树的（事实上，根据中序遍历和后序遍历也可以）。前序遍历和后序遍历无法重现原二叉树。
 在上文中，已经介绍了各种遍历的方式。对于前序遍历，它的特征是先访问根结点，再访问左子树，最后访问右子树。对于中序遍历，它是先访问左子树，再访问根结点，最后访问右子树。这两种遍历方式的特征，构成了重建的基础：

  1. 先序遍历的结果：根结点在第一位，然后依次是左子树和右子树
  2. 中序遍历的结果：左子树在左边，以根结点为分界线，右边是右子树
  3. 通过先序遍历，确定了根结点的值，在通过中序遍历的结果，确定左子树集合和右子树集合
  4. 左子树和右子树的集合，本质上也是对左子树和右子树经过先序遍历、中序遍历出来的结果。符合第 1 、2 、 3 所描述的特征。因此，这件事可以通过递归的方式解决。

小结

 这一节，其实涉及到了分治递归、栈、队列、树，是算法基础的一种集中体现。良好的算法基础，将非常有助于我们处理计算机问题。
