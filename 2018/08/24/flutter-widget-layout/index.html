创建 layout

Step 0: 开始

  • 首先需要确认你已经搭建好了环境
  • 创建一个基本的 Flutter app

 接着，在实例中添加一个图片：

  • 在项目的根目录下创建一个 images 文件夹
  • 添加 lake.jpg
  • 更新 pubspec.yaml 文件，使其包含 assets 标签，这样你的 image 才可以在代码中使用。

Step 1：图解布局

 第一步，就需要将整个布局拆解成基本元素：

  • 确定行和列
  • 布局中包含网格（grid）吗？
  • 有重叠的元素吗？
  • UI 中需要 tab 吗？
  • 注意对齐方式、padding 和 border

 首先，来确定最大的元素，在这个实例中，有四个元素排成一列：一个 image，两个 row ，一个 text 块。

 接下来，图解每一行。第一行，称为标题区域，拥有三个孩子：一列 text、一个 star icon 和一个数字。第一个孩子，是一个 column （列），包含两行 text。这个列占据了更多的空间，因此需要被包裹在一个 Expanded widget 中。
 第三行，是按钮部分。这一部分有三个 children : 每个 child 是一个列，包含了 icon 和text。

 一旦布局被图解以后，就很容易采用自下而上的方法实现它。为了避免过深的嵌套，可以使用一些变量和 function。

布局方法

观点：

  • widget 是用来构建 UI 的类
  • widget 被用来布局和 UI 元素
  • 复杂的 widget 是由简单的 widget 组合而成的

 Flutter 布局机制的核心是 widget。在 flutter 中，几乎所有东西都是 widget，比如 image、icon、text 等所有你在 Flutter app 中看到的都是 widget。但是，你看不到的，例如 row、column、grid 等这些用来排列、约束、对齐可见 widget 的其实也是 widget。
 你可以组合 widget 从而构建一个复杂的 widget。比如说，底下第一张的截图中，展示了 3 个下面带有标签的 icon：


 上图中，显示了一个可见的 layout，3 个 column 按 row 排列，每个 column 中包含一个 icon 和一个label。

 上图中，大致应该和你想的一样，但是，你可能会好奇那个 Container(标记为 粉色的)。 Container 是一个允许你自定义自己 child widget 的 widget。使用 Container 你可以设置 padding、margin、border、背景色等。
 在这个示例中，每个放置在 Container 中的 Text widget 添加了 margin 属性。整个 Row 被放置在一个 Container，是为了设置其 padding 属性。
 本例中剩下的部分，就是给 UI 设置属性。使用 color 属性给 Icon 上色，使用 style 属性给 Text 设置字体大小、颜色、字体等。Column 和 Row 允许你指定子控件的排列方式、占位空间等。

放置 widget

观点：

  • 甚至连 App 本身就是一个 widget
  • 创建一个 widget 并将其添加到 layout widget 中是非常容易的
  • 为了将 widget 显示在设备上，需要将 layout widget 添加到 app widget 中
  • 最简单的是使用来自 Material Component 库中的 Scaffold，它提供了一个默认的 banner，背景色，同时又包含添加 drawer、snack bar 和 bottom sheet 的 api
  • 如果你喜欢，可以仅仅使用标准库中的 widget 来构建你的 app
