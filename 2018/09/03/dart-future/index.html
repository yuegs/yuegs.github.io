异步编程：Futures [译]

重点：

  • Dart 是单线程的
  • 同步代码可能会导致程序卡死
  • 使用 Futures 执行异步操作
  • 在异步函数中，使用 await 暂停当前代码的执行，直到 Future 完成
  • 或者使用 Future 的 then() 方法（跟上面的效果一样）
  • 在异步方法中，使用 try-catch 表达式捕获异常
  • 或者使用 Future 的 catchError() 方法
  • 你可以链接 Future 以执行异步函数

 Dart 是一个单线程编程语言。如果代码阻塞了线程的执行（比如一些耗时操作），程序可能会被卡住。异步操作可以帮助你避免这类事件发生。Dart 使用 Future 对象代表异步操作。

 为了保证应用的响应速度，Dart 库允许在定义函数时，使用异步模型来完成潜在的耗时操作。这些函数会使用 Future 返回 value。

Future 是什么

 Future 代表了一个可以返回值（可能是 void）的异步操作。当一个返回 Future 的函数被调用，会有两件事发生：

  1. 将任务入列，返回一个未完成的 Future(uncompleted future)
  2. 当操作完成之后，Future 以一个 value 或者 error 完成

 一个 Future 会以两种方式完成：正常返回一个 value (with a value)或者抛出异常(with an error)，对此我们都可以注册回调来处理。
 特别的，如果在注册函数中，返回的 value 是一个 Future，则先前 Future 将等待 返回的新的 Future 一起完成。它们将返回同样的结果。
 但使用 Future 写代码的时候，有如下两种选择：

  • 使用 async和 await
  • 使用 Future API

async 和 await

 async 和 await 关键字是 Dart 语言中，asynchrony support 的一部分。它们允许你像写同步代码一样写异步代码，而不使用 Future API. async 函数 是在函数体前以 async 关键字标记。而 await 关键字仅仅在 async 函数中使用。

 下面以图解的方式，展示执行过程。

  1. app 开始执行
  2. main() 函数开始执行同步代码，第一行就进行异步调用 printDailyNewsDigest()
  3. printDailyNewsDigest() 使用 await 调用了 gatherNewsReports(),此时开始执行异步代码
  4. 因为 getherNewsReports() 是一个异步函数，当它执行到 return 语句时，它暂停了执行，并返回了一个 uncompleted Future（一个 Future 实例）
  5. 因为 printDailyNewsDigest() 是一个异步函数，它暂停了执行，返回一个 uncompleted Future（在本例中，是一个 Future 实例） 给它的调用者 main()
  6. 接下来的打印工作继续。因为它们是同步代码，接下来将按照顺序打印
  7. 当 main() 执行完，异步代码得以重新执行。首先，被 getherNewsReports() 返回的 Future 得以完成。接着，printDailyNewsDigest() 执行打印工作
  8. 当 printDailyNewsDigest() 函数体执行按成，原来的 Future 完成，app 退出。

 注意，当一个异步函数遇到下面执行语句时，将暂停执行，并返回一个 uncompleted Future：

  • 函数中，第一个 await 表达式
  • 函数中的 return 语句
  • 函数体执行完了
