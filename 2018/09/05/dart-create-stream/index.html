在 Dart 中创建 Stream [译]

发表于 2018-09-05 | 分类于 Dart |
字数统计: 2,110

 原文链接
 在 dart:async 库中，有两个用途广泛的 Dart API : Stream 和 Future。Future 代表着一个计算结果，而 Stream 代表着一系列的结果。你可以在 Stream 设置监听，以等待结果（这个结果可能是 data ，也可能是 error）的到来。你也可以暂停或者停止对 Stream 的监听。
 但是，本篇文章不是关于如何使用 Stream，而是关于如何创建 Stream。Stream 的创建方式有很多，可以是根据一个已经存在的 Stream 做转换创建一个新的 Stream，也可以使用 StreamController ，如果需要，你还可以继承 Stream。这篇文章，将通过代码展示这些创建方式。关于如何使用部分，可以查看 Asynchronous Programming: Streams

Stream 是什么

 译者注：在此之前，我想先来了解下 Stream 是什么，对 Stream 有一个大致的了解再来谈创建。本段同样来自于官网中的另一篇 。
 Stream 是异步数据事件源。它提供了一种接收顺序事件的方式。每一个事件是一个数据事件（也被称为 stream 中的元素），或者是一个异常事件，提示有些事失败了。
 这里有两种 Stream：单次订阅 stream 和广播stream。
 单次订阅 stream，在整个 stream 生命周期内仅允许单个 listener。直到它有了一个 listener，它才开始生产事件，当 listener 不在订阅事件的时候，就停止发送事件，即使事件源仍然在提供更多的事件。
 在同一个单次订阅 stream 上监听两次是不允许的，即使第一次订阅已经取消了。
 单次订阅 stream 通常用于比较大的持续的数据流中，比如 IO。
 广播 stream 允许任意多个 listener。当事件准备好之后，它就发射事件，不管此时有没有 listener 存在。
 广播流通常应用于观察者模式下。
 如果多个 listener 想监听一个 单次订阅 stream，可以使用 asBroadcastStream 来在这个 non-broadcast stream 上创建一个 broadcast stream。
