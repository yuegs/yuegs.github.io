事件驱动中的基础：队列

发表于 2019-01-09 | 分类于 Android |
字数统计: 1,181


GUI 程序的设计通常都是基于 事件驱动 的。比如说 Android App、Dart 客户端 (如上图所示) 、NWJS 客户端等，还有一些用于异步处理的库，比如 RxJava 中。事件驱动通常和队列是联系在一起，不断有外部（系统）或者内部事件加入到事件队列中，不断有事件从队列中取出并得到处理。
当有很多事件同时涌入，队列的出现，正是为了解决了事件生产者和消费者之间处理速度不统一的矛盾，使得每一个事件都的按照顺序依次执行。
接下来，我想讨论下在 Dart 中 、Android 中的队列处理模型，最后，来看下 RxJava 中队列的具体实现。最终的目的，是希望在需要的时候自己能够设计出来这样一套模型。

Dart: Evnet Queue & Microtask Queue

在 Dart 中，使用了两种队列，Event Queue 和 Microtask Queue。后者拥有较高的优先级，即当 Microtask Queue 为空的时候，才考虑执行 Evnet Queue 中的事件。

Android：

在 Android 中，自然也存在类似的事件队列。用户点击、滑动等各种手势，还有各种需要线程之间的通信，也需要这样一个 Looper。

上图来源于Android Handler Tutorial
在 UI 线程中，维护这样一个 Looper。其他线程通过 Handler 将 Message 发送到于主线程（当前也可以是其他线程）所关联的 MessageQueue 中，主线程会从中取出消息进行处理（执行 msg.callback.run()）。Handler-Message-MessageQueue-Thread 之间存在着对应关系。Thread 可以关联一个 MQ,MQ 可以关联很多 Handler，其所关联的 Handler 会将 Message 发送到该 MQ 中。
