在 Flutter 中使用 Firebase Cloud Messaging

三、小结

  1. 消息的发送，可以直接通过 Firebase 控制台发送，也应该可以接入到自己的后台消息服务中，使用相关SDK，根据业务逻辑发送。后者的应用场景更多，更灵活。
  2. 消息的接收有两种情况，可以通过 package name 进行消息群发，也可以通过 token 做消息的针对性发送。
  3. 消息的达到，根据目前手头上的 honor 9 来看，杀掉进程后，无法接收到消息，那么，在需要确保消息到达的情况下，在接收到消息后，可以给后台一个反馈。
  4. 消息到达后，APP 处于正在交互或者处于后台运行时，需要做不同的消息提醒方式。
  5. 消息的到达，存在一定延时，从目前来看，延时不高，目前最迟没有发现超过5分钟的。
  6. Firebase 不仅可以从后台往 APP 发送消息，反之亦可。还可以做统计分析。
