一种原地归并算法

 所谓归并，就是将两个已经排好序的序列合并成一个较大的序列。而归并排序算法，就是建立在此基础之上。在具体实现方向上，我找到了两个方式。他们都称为原地归并。

in-place algorithm is an algorithm which transforms input using no auxiliary data structure. However a small amount of extra storage space is allowed for auxiliary variables. The input is usually overwritten by the output as the algorithm executes. In-place algorithm updates input sequence only through replacement or swapping of elements.

 原地，就是在输入空间的基础上进行操作输出，有时会使用额外的辅助空间。

 前文中，我们已经知道，从用途和生存期的角度看注解可以分为三类:

  • 编译器指令，用于告诉编译器一些信息，比如 @Override 等，帮助我们纠错。这类注解在生存期上对应于 RetentionPolicy.SOURCE ，在编译的时候，会被编译器丢弃。
  • 编译期指令，运行在编译过程中，可以用来生成类等文件；在生存期上对应于 RetentionPolicy.CLASS ，在编译期存在，但是在虚拟机运行时注解将不被保留。注意，如果不显示指定 Retention 的值，该值将作为默认值。
  • 运行期指令，主要用于反射。在生存期上对应于 RetentionPolicy.RUNTIME ，在编译时和运行时，注解均存在。

 除了编译器指令，后两者都需要一个处理器能够处理它们。今天，就来讲讲这些处理器的自定义过程。
