 前文中，我们已经知道，从用途和生存期的角度看注解可以分为三类:

  • 编译器指令，用于告诉编译器一些信息，比如 @Override 等，帮助我们纠错。这类注解在生存期上对应于 RetentionPolicy.SOURCE ，在编译的时候，会被编译器丢弃。
  • 编译期指令，运行在编译过程中，可以用来生成类等文件；在生存期上对应于 RetentionPolicy.CLASS ，在编译期存在，但是在虚拟机运行时注解将不被保留。注意，如果不显示指定 Retention 的值，该值将作为默认值。
  • 运行期指令，主要用于反射。在生存期上对应于 RetentionPolicy.RUNTIME ，在编译时和运行时，注解均存在。

 除了编译器指令，后两者都需要一个处理器能够处理它们。今天，就来讲讲这些处理器的自定义过程。

高德地图——让 Marker 在规划好的路线上平滑移动

发表于 2018-04-22 | 分类于 Android | | 阅读次数:
字数统计: 2,852

 Marker，即地图上的覆盖物；规划好的路线，即在若干关键点之间进行路径规划（步行、骑行、驾车）；平滑移动，即按照一定速度移动，并且可以动态实时调整 Marker 的移动速度。
 对于 marker 来说，本质上就是动态在界面上添加一个 View，所以这里需要注意的是，添加的顺序会导致不同 marker 的遮挡问题。所谓规划好的路线，实际上就是一条折线。而这个折线，是其先在路线上采点，然后用直线连接相连两个点之后形成的。那就是如果对于 A 点 和 B 点之间，物理上原本就是一条直线，那么它采点可能就是稀疏的，而如果 A 和 B 之间是一条弯弯曲曲的路径，为了能更好的描述这个细节，这段路径上采点可能就会密集。相邻两个点之间，就是一个 Step，可以封装这段路程之间的距离、需要走多长时间等信息。平滑移动，运行着的是一个动画。我们都知道，动画是一帧一帧的，本质上来讲，其实就是我们在极短的时间里不断改变 marker 的位置，就形成了移动。速度的改变，其实就是调节这段时间内，marker 位置变化的程度。
